# AIRONE Website Build Order List

## Purpose

This document defines the full AIRONE website build order, page priority, launch requirements, completed pages, pending pages, and client inputs required before publishing.

The build should follow the Site Master Framework and preserve continuity with the existing AIRONE static HTML site.

Primary direction:

Keep the same existing page style, layout rhythm, visual identity, section structure, CTA style, navigation style, footer style, and content tone across all new and updated pages.

This is not a redesign.

This is a continuity-based build and optimization process.

---

# Build Rules

## Global Build Rules

1. Keep the existing AIRONE page style and structure.
2. Keep existing URLs stable unless a documented URL issue requires a change.
3. Do not redesign the site from scratch.
4. Do not introduce a new design system.
5. Do not change the brand identity.
6. Do not invent reviews, ratings, awards, license claims, guarantees, pricing, warranties, or years in business.
7. Mark unknown business facts as TODO.
8. FAQ schema must match visible FAQ text exactly.
9. Do not add AggregateRating or Review schema unless verified data exists.
10. Update sitemap.xml when new pages are added.
11. Check robots.txt, _headers, and _redirects when URL or hosting behavior changes.
12. Preserve static HTML cleanliness and valid markup.
13. Use consistent internal linking between related service pages.
14. Prioritize revenue pages before city pages and blog content.
15. Nothing should go live until Phase 1 revenue pages are complete.
